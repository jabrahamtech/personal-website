Architecture

Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore. The flow is built as a node graph rather than a single LLM prompt — each node is a small, testable unit with explicit inputs, outputs, and side-effects.

  • Telephony — Twilio Media Streams over WebSocket; we never touch the audio file at rest.
  • ASR — streaming transcription with partial results, debounced for endpointing.
  • LLM core — Groq for latency, with structured-output schema enforced server-side.
  • State machine — required-fields tracker; the LLM cannot leave the call without them.
  • Memory — short-term scratchpad + long-term key-value store, both auditable.
  • Summary & handoff — at hangup, a structured payload is emitted to the CRM with retries.
