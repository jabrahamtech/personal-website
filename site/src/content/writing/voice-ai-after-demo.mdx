## The four things demos hide

### 1. State across turns

### 2. Interruptions

Real callers interrupt. They cough. They turn to talk to someone else. They hang up mid-sentence. Every node in your flow has to be cancellable — not "throw an exception", **cancellable**. Most demos can't even pause cleanly.

### 3. Memory you can audit

### 4. Handoff

The voice agent is rarely the system of record. It's a collector. The handoff to the CRM, the ticketing system, or the human queue has to be **idempotent, retryable, and provenance-tagged**. Most demos handwave this with a webhook.

## If you're building one

1. Write the call as a state machine on paper before you touch a model.
2. Make every node cancellable. Test by hanging up at every step.
3. Log everything the model said and why. Every field needs a source.
4. Treat handoff as a separate, retryable system. Not a callback.
5. Demo with hostile callers, not friendly ones.
