## Draft angle

Not every step in an AI agent should be handled by AI. Most production agents work better when deterministic software owns the boring parts: validation, routing, retries, permissions, state transitions, and audit trails.

## Intended reader

This is for teams planning an agent build and trying to decide what belongs in prompts, what belongs in tools, and what belongs in traditional application code.

## Decision points

- Use rules when the desired behaviour is stable and auditable.
- Use queues when work can fail, retry, or need human escalation.
- Use database constraints when correctness matters more than model flexibility.
- Use model reasoning when the input is ambiguous enough that fixed logic becomes brittle.
